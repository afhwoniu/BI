# 开发任务拆分与迭代规划

> 目标：在保证架构合理的前提下，按阶段、可迭代地实现决策分析 BI 可视化系统。  
> 优先打通主链路：**数据源 → 数据集 → 图表 → 面板**，再逐步增强高级能力。

---

## 1. 阶段划分概览

| 阶段 | 名称 | 核心目标 |
|------|------|----------|
| 阶段 0 | 项目初始化与基础设施搭建 | 搭建前后端工程、管理库、认证基础 |
| 阶段 1 | 核心链路 MVP | 数据源 → 数据集 → 图表 → 简单分析面板 |
| 阶段 2 | 高级分析能力 | 筛选、外链参数、下钻联动、全局筛选、同环比 |
| 阶段 3 | 报表/报告与集中展示 | 报表编辑、菜单、发布与权限 |
| 阶段 4 | 多终端与性能优化 | 大屏、移动端、缓存、备份还原 |

---

## 2. 阶段 0：项目初始化与基础设施

### 2.1 后端项目搭建（ASP.NET Core）

## 3. 阶段 1：核心链路 MVP

> 目标：从"配置数据源 → 定义数据集 SQL → 预览 → 配置图表 → 放入面板"这条主链路打通。

### 3.1 数据源管理

| 任务编号 | 任务名称 | 说明 |
|----------|----------|------|
| 1-1 | 完善 `bi_datasource` 表结构 | 字段：`id`、`name`、`type`、`conn_string`、`remark`、`is_enabled`、`created_at` 等 |
| 1-2 | 后端数据源管理 API | 新增/编辑/删除/列表/详情/测试连接 |
| 1-3 | 前端数据源管理页面 | 列表展示、弹窗编辑、测试连接按钮 |

### 3.2 数据集管理（SQL 数据集）

| 任务编号 | 任务名称 | 说明 |
|----------|----------|------|
| 1-4 | 完善 `bi_dataset` / `bi_dataset_field` 表结构 | 参数定义 JSON、字段角色（dim/measure）、聚合类型 |
| 1-5 | 后端数据集 CRUD 与预览 API | 创建/编辑、预览（安全校验+执行 SQL+返回字段元数据） |
| 1-6 | 前端数据集管理页面 | SQL 编辑器、参数配置、预览数据、字段列表（标记维度/度量） |

### 3.3 图表管理与执行

| 任务编号 | 任务名称 | 说明 |
|----------|----------|------|
| 1-7 | 设计 `bi_chart` 表 | 字段：`id`、`name`、`dataset_id`、`type`、`config_json`、`remark` 等 |
| 1-8 | 后端图表 CRUD 与查询 API | 创建/修改/删除/列表；`POST /charts/{id}/query` 构建聚合 SQL 并执行 |
| 1-9 | 前端图表设计器页面 | 左栏字段列表、中间拖拽区、右栏样式配置、实时预览 |

### 3.4 简单分析面板（Dashboard MVP）

| 任务编号 | 任务名称 | 说明 |
|----------|----------|------|
| 1-10 | 设计 `bi_panel` / `bi_panel_item` 表 | 面板主表、子项表（chart_id + layout_json） |
| 1-11 | 后端面板 CRUD + 渲染 API | CRUD、获取面板详情及子项布局 |
| 1-12 | 前端面板编辑器（简单版） | 从图表列表拖拽到画布、网格布局调整、保存布局 |

### 3.5 阶段 1 验收标准

- [x] 能创建数据源，并成功测试连接
- [x] 能基于数据源创建数据集，编写 SQL 并成功预览数据
- [x] 能基于数据集创建图表，配置维度/度量并成功展示图形
- [x] 能创建分析面板，把多张图表放入其中并展示

## 4. 阶段 2：高级分析能力

> 目标：让平台从"能用"进一步升级为"好用"，重点在分析交互。

### 4.1 筛选与外链参数

| 任务编号 | 任务名称 | 说明 |
|----------|----------|------|
| 2-1 | 图表预筛选配置 | 在 `config_json` 中加入预筛选定义；后端查询时转为 where 子句 |
| 2-2 | 外链参数支持 | 图表/面板配置 URL 参数映射；请求时从 URL 取参数注入筛选 |
| 2-3 | 分析面板本体筛选 | 面板级配置本体筛选器，前端提供筛选 UI，变化时刷新子图表 |

### 4.2 全局筛选与下钻联动

| 任务编号 | 任务名称 | 说明 |
|----------|----------|------|
| 2-4 | 全局筛选定义 | 在 `bi_panel.config_json` 定义全局筛选项，配置响应图表 |
| 2-5 | 后端全局筛选支持 | 图表查询接口接受面板级过滤条件，映射到字段并加入 where |
| 2-6 | 下钻联动配置与实现 | `bi_panel_link` 表存储联动关系；前端 onClick 触发目标图表刷新 |

### 4.3 同环比分析

| 任务编号 | 任务名称 | 说明 |
|----------|----------|------|
| 2-7 | 图表同环比配置 | 仅日期维度图表开放，支持同比/环比开关 |
| 2-8 | 后端同环比计算 | 推导对比区间，一次查询返回当前期和对比期数据 |

## 7. 优先级与建议路线

### 7.1 强烈建议优先完成

**阶段 0 + 阶段 1**：打通"数据源 → 数据集 → 图表 → 面板"的核心体验，形成可演示的 MVP。

### 7.2 次优先级

**阶段 2**：加入筛选、外链参数、下钻、全局筛选、同环比，使分析能力更加接近 DataInsight 使用体验。

### 7.3 中期建设

**阶段 3**：报表/报告 + 菜单 + 发布，使系统真正可以"对外统一展示"。

### 7.4 后期优化

**阶段 4**：针对大屏、移动端、性能与运维持续优化，逐步企业级落地。
